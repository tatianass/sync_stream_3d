# Sync Stream 3D

A high-performance producer-consumer engine designed to synchronize, validate, and convert robotics data from **MCAP** files into a training-ready **Parquet + Binary** dataset.

## 🚀 Overview

This engine streams synchronized LiDAR and Camera messages from MCAP files, applies data quality guards via **Great Expectations**/**Pydantic**, exports structured datasets and offers a **Pytorch** custom dataset for training.

### Key Features:
* **Synchronized Streaming:** Pairs LiDAR point clouds and Image frames using exact timestamp matching.
* **Multi-threaded I/O:** Uses `ThreadPoolExecutor` to offload image encoding and disk writes.
* **Quality Guard:** Automatic data validation (null checks, range checks, regex tagging, schema) before final indexing.
* **Dockerized Environment:** Fully reproducible builds using `uv` and Debian Bookworm-Slim.

## 📂 Project Structure

* `main.py`: Entry point for the pipeline.
* `data/`: Input directory for raw `.mcap` files.
* `output/`: Generated datasets including `/images`, `/lidar`, and `index.parquet`.
* `syncstream/`: Core logic modules:
    * `chunker.py`: Handles MCAP streaming and message synchronization.
    * `converter.py`: Manages the transformation and saving of synchronized chunks.
    * `validation.py`: Great Expectations implementation for data quality.
    * `tagging.py`: Logic for semantic metadata and lighting classification.
    * `dataset.py`: A custom Pytorch Dataset class for training.
    * `schemas.py`: Used for Pydantic checking.
    * `constants.py`: Where constants are defined.
    * `utils.py`: Utility functions.
* `tests/`: Unit and integration test suite.
* `visualizer.py`: Utility to verify synchronized data streams.

## 📚 The Dataset
The `SyncStream3DDataset` class in `syncstream/dataset.py` is a custom PyTorch implementation designed for high-performance training. It provides synchronized access to **Pose**, **LiDAR point clouds**, and **Camera images**.

### Basic Usage
```python
from pathlib import Path
from syncstream.dataset import SyncStream3DDataset

# Initialize using the Parquet index generated by the pipeline
dataset = SyncStream3DDataset(index_file=Path("output/index.parquet"))

# Access a synchronized sample
sample = dataset[50]

print(f"Timestamp: {sample['timestamp']}")
print(f"Image Shape: {sample['image'].shape}")  # RGB Camera frame
print(f"LiDAR Data: {sample['lidar'].shape}")  # NPY point cloud
print(f"Pose Tensor: {sample['pose']}")        # [x, y, z, qx, qy, qz, qw]

### Automated Tagging & Filtering

```python
import pandas as pd

# Load the index and filter for relevant tags (e.g., 'night' or 'discontinuous')
df = pd.read_parquet("output/index.parquet")
night_samples = df[df['semantic_tags'].str.contains("night")]

# Load the filtered subset into the training pipeline
night_dataset = SyncStream3DDataset(index_file=Path("output/index.parquet"))
night_dataset.df = night_samples
```

## 🧠 Design Decisions & Heuristics

### Temporal Synchronization
* **Threshold (150ms):** The engine uses a 150ms maximum delta for pairing LiDAR and Camera frames. This ensures spatial consistency at standard robotic traversal speeds (e.g., at 1.4m/s walking speed, max displacement is <0.2m), maintaining the geometric integrity of the sensor fusion.
* **Producer-Consumer Architecture:** To prevent I/O bottlenecks, the MCAP streamer (Producer) and the File Encoder (Consumer) run in parallel, ensuring that the heavy JPEG/NPY serialization doesn't stall the data ingestion.

### Data Quality & Tagging
* **Luminance Analysis:** Automated tagging for "night/dark" scenes uses an RMS (Root Mean Square) pixel intensity threshold, allowing for easy dataset filtering for low-light training.
* **Motion Guard:** Uses frame-to-frame variance to tag "high-motion" blur, helping to exclude low-quality samples from the training set.
